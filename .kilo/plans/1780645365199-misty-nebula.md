# Rust Application Plan: Crypto/Forex Price Tracker with ODS Output

## Overview
Create a Rust application that fetches data from 5 APIs and outputs to a LibreOffice Calc (.ods) file with timestamp.

## Dependencies (Cargo.toml)
- `reqwest` - HTTP client for API calls
- `serde` + `serde_json` - JSON parsing
- `tokio` - async runtime
- `spreadsheet-ods` - ODS file read/write support
- `chrono` - timestamp formatting

## API Endpoints Analysis

### API 1: CryptoCompare Price Multi (USD)
- URL: `https://min-api.cryptocompare.com/data/pricemulti?fsyms=BTC,ETH,DOGE,TRX,ADA,NIGHT,BDAG,USDT,USDC&tsyms=USD`
- Response: `{ "BTC": {"USD": price}, "ETH": {"USD": price}, ... }`
- Extract all USD prices

### API 2: CryptoCompare Price Multi (BTC)
- URL: `https://min-api.cryptocompare.com/data/pricemulti?fsyms=ADA,NIGHT,BDAG,TRX,DOGE,BNB,ETH,USDT,USDC&tsyms=BTC`
- Response: `{ "ADA": {"BTC": price}, ... }`
- Extract BTC prices

### API 3: CryptoCompare Price (ZAR)
- URL: `https://min-api.cryptocompare.com/data/price?fsym=BTC&tsyms=ZAR&e=VALR`
- Response: `{ "ZAR": price }` - BTC price in ZAR (e.g., ~1028606 ZAR ≈ $62860)
- Extract ZAR price

### API 4: OpenExchangeRates Latest
- URL: `https://openexchangerates.org/api/latest.json?app_id=3263b0c93523446299d17e2e6abdd748&symbols=ZAR,THB,KZT`
- Response: `{ "rates": {"ZAR": rate, "THB": rate, "KZT": rate} }`
- Extract **only** rates values (ZAR, THB, KZT)

### API 5: VALR USDTZAR Market Summary
- URL: `https://api.valr.com/v1/public/USDTZAR/marketsummary`
- Response: JSON object with market data
- Extract: **only** lastTradedPrice

## Implementation Structure

### `src/main.rs`
1. Define data structures for API responses using `serde`
2. Create async functions for each API call
3. Fetch all data concurrently (using `tokio::join!`)
4. Combine results into a flat row structure
5. Write to ODS file with timestamp

### Data Output Format
**ODS Sheet Name:** CryptoPriceData  
**Output File:** `CryptoPriceData.ods` (in current working directory)

**ODS Sheet Columns:**
1. Timestamp (YYYY-MM-DD HH:MM)
2. BTC_USD, ETH_USD, DOGE_USD, TRX_USD, ADA_USD, NIGHT_USD, BDAG_USD, USDT_USD, USDC_USD (from API 1)
3. ADA_BTC, NIGHT_BTC, BDAG_BTC, TRX_BTC, DOGE_BTC, BNB_BTC, ETH_BTC, USDT_BTC, USDC_BTC (from API 2)
4. BTC_ZAR (from API 3)
5. ZAR_XR, THB_XR, KZT_XR (from API 4 - exchange rates, base USD)
6. USDTZAR_lastTradedPrice (from API 5)

### Error Handling
- Handle HTTP request failures
- Handle missing/malformed JSON fields
- Log errors but continue with available data

## Execution Flow
1. Check if `CryptoPriceData.ods` exists
2. Fetch all 5 API responses concurrently
3. Parse JSON responses into structured data
4. If file does not exist: create new ODS file with headers and write data row
5. If file exists: append data row to existing file
6. Save ODS file

## Notes
- File name must be exactly `CryptoPriceData.ods`
- API 4 requires careful extraction of nested `rates` object
- All values (except timestamp) are outputted as float numbers for proper numeric handling in LibreOffice Calc
- Recommended ODS library: `spreadsheet-ods` - supports read/write operations for appending
